================================================================================
                           WHAT EACH FOLDER DOES
================================================================================

app/
  ├─ 📌 ROOT APPLICATION FOLDER
  ├─ Contains all pages and layouts
  ├─ Each subfolder = one route
  ├─ page.tsx = the page component
  └─ Uses Next.js 16 App Router pattern

app/context/
  ├─ 📌 STATE MANAGEMENT
  ├─ AuthContext.tsx provides global auth state
  ├─ useAuth() hook for accessing auth
  └─ Login/logout functions and user data

components/
  ├─ 📌 REUSABLE UI COMPONENTS
  ├─ Not tied to specific pages
  ├─ Imported and used by pages
  ├─ Examples: Sidebar, ProtectedRoute, LoadingSpinner
  └─ Easy to reuse across multiple pages

lib/
  ├─ 📌 UTILITIES AND HELPERS
  ├─ api.ts: Axios HTTP client with endpoints
  ├─ types.ts: TypeScript type definitions
  └─ Used throughout the app

public/
  ├─ 📌 STATIC ASSETS
  ├─ Images, fonts, favicon, etc.
  ├─ Served directly without processing
  └─ Build into optimized output

.next/
  ├─ 📌 BUILD OUTPUT (Auto-generated)
  ├─ Created by: npm run build
  ├─ Contains production-optimized files
  ├─ Don't edit this folder
  └─ Can be safely deleted and rebuilt

node_modules/
  ├─ 📌 DEPENDENCIES (Auto-generated)
  ├─ Created by: npm install
  ├─ Contains all npm packages
  ├─ Don't edit this folder
  └─ Can be safely deleted and reinstalled

================================================================================
                          HOW FILES RELATE
================================================================================

AUTHENTICATION FLOW
└─ app/login/page.tsx (form)
   └─ calls → lib/api.ts (login endpoint)
      └─ stores token → app/context/AuthContext.tsx
         └─ provides auth to → app/providers.tsx
            └─ wraps → app/layout.tsx
               └─ used by → components/ProtectedRoute.tsx
                  └─ guards → all other pages

PAGE STRUCTURE
├─ app/[page]/page.tsx (page component)
├─ wrapped by → components/ProtectedRoute.tsx
├─ wrapped by → components/DashboardLayout.tsx
├─ contains → components/Sidebar.tsx
└─ fetches data → lib/api.ts
   └─ uses types → lib/types.ts
   └─ uses styles → app/globals.css

STYLING FLOW
├─ Tailwind CSS → tailwind.config.ts
├─ Global styles → app/globals.css
├─ CSS variables → app/globals.css (:root)
├─ Component classes → app/globals.css (@layer)
└─ Applied to → All components and pages

================================================================================
                           FIRST TIME SETUP
================================================================================

1. FILES ALREADY EXIST - JUST INSTALL & RUN!

   $ npm install                  # Downloads dependencies
   $ cp .env.local.example .env.local  # Copy environment template
   $ npm run dev                  # Starts development server

2. WHAT HAPPENS:
   
   npm install:
   ├─ Reads package.json
   ├─ Downloads packages from npm
   ├─ Creates node_modules/ folder
   └─ Creates package-lock.json (don't edit)

   npm run dev:
   ├─ Starts Next.js dev server
   ├─ Creates .next/ folder
   ├─ Hot reloads on file changes
   └─ Opens http://localhost:3000

3. YOU'RE READY!
   
   All files are already written and waiting
   Just install dependencies and run!

================================================================================
                      WHEN YOU WANT TO...
================================================================================

ADD A NEW PAGE

  1. Create folder: app/my-page/
  2. Add file: app/my-page/page.tsx
  3. Start with template code
  4. It automatically becomes a route: /my-page

ADD A COMPONENT

  1. Create file: components/MyComponent.tsx
  2. Export default function
  3. Import in pages that need it
  4. Use it like: <MyComponent />

ADD AN API ENDPOINT

  1. Open lib/api.ts
  2. Add to appropriate API section (authAPI, appointmentAPI, etc.)
  3. Use pattern: export const myEndpoint = () => apiClient.get('/path')
  4. Import and use in pages

CHANGE COLORS

  1. Open app/globals.css
  2. Find :root { ... }
  3. Change color values (e.g., #2563eb)
  4. Auto-updates everywhere

CHANGE STYLING

  1. Use Tailwind classes in JSX (className="...")
  2. Or add custom CSS classes in app/globals.css
  3. Use CSS custom properties from :root
  4. Examples in CODE_REFERENCE.md
